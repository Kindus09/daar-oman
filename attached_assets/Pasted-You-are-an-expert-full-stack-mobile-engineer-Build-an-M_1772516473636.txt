2) Broker/Agent
3) Admin (minimal; simplest implementation is an “admin mode” screen for allowlisted admin phone numbers)

MVP REQUIREMENTS (MUST HAVE)

AUTH & ONBOARDING
- Phone OTP sign-in for all users
- First login: choose role (Requester or Broker)
- Profiles:
  - Requester: name, phone
  - Broker: name, phone, optional agencyName, serviceAreas[], propertyTypes[], rentBuyModes[] (rent/buy/both)

REQUESTS (Requester)
- Create a Request with REQUIRED fields:
  - transactionType: Rent or Buy
  - propertyType
  - area (free text OR simple list)
  - budget (number, OMR)
- My Requests list with status (Open/Closed)
- Request detail:
  - list Offers
  - close request with outcome: rented/bought/not interested/still looking

BROKER DISCOVERY
- Broker Home has 2 tabs:
  1) Matched Requests (simple matching using brokerProfile filters; keep simple)
  2) All Requests feed with filters (transactionType, propertyType, area; budget optional)
- Broker can open request detail and send offer

OFFERS
- Offer creation:
  - required message text
  - optional link URL
  - optional price number
- Brokers can send either “I have options, let’s talk” or include link/details

CHAT
- One thread per (Request, Requester, Broker)
- Realtime Firestore chat: message list + input + send

CONTACT ACTIONS
- Call button -> opens dialer with number
- WhatsApp button -> deep link with number
- No in-app calling, no file sharing

NOTIFICATIONS
- Requester: new offer + new message
- Broker: new message (+ optional new matched request)

SAFETY / MODERATION
- Requester can Block and Report broker
- Blocking prevents broker from sending offers/messages to that requester
- Admin tools:
  - list users/requests/offers/reports
  - disable user
  - remove request/offer
Implement admin by checking if logged-in phone is in ADMIN_PHONE_ALLOWLIST (env var).

FIRESTORE DATA MODEL
- users/{userId}
- requests/{requestId}

- threads/{threadId}
- threads/{threadId}/messages/{messageId}

SECURITY RULES
- Users: only self profile

- Offers: brokers create; requesters read offers for their requests
- Messages: only thread participants
- Blocks: enforced to prevent offers/messages
- Admin override: allowlisted admins can moderate
